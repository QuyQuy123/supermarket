QUY TRINH LAM VIEC GIT THEO TEAM (MOI THANH VIEN LAM VIEC TREN BRANCH RIENG)
==========================================================================

MUC TIEU
--------
- Moi thanh vien code tren branch rieng, KHONG code truc tiep tren main.
- Main luon on dinh, moi thay doi di qua Pull Request (PR).
- Giam xung dot, de review, de rollback khi can.


0) YEU CAU CHUNG
----------------
- Dat ten branch ro rang:
  + feature/<mo-ta-ngan>
  + bugfix/<mo-ta-ngan>
  + hotfix/<mo-ta-ngan>
  + chore/<mo-ta-ngan>
- Commit message ro nghia, ngan gon.
- Truoc khi push/PR: chay test, lint, build co ban.
- Khong commit file may ca nhan (.idea, .vscode local, env secrets, build cache...).


1) LAN DAU LAY PROJECT VE MAY
-----------------------------
git clone https://github.com/ten-repo/project.git
cd project

Kiem tra remote:
git remote -v


2) KIEM TRA NHANH TRANG THAI REPO
---------------------------------
git branch
git status

Neu dang khong o main:
git checkout main


3) CAP NHAT MAIN MOI NHAT (BAT BUOC TRUOC KHI BAT DAU TASK)
------------------------------------------------------------
git checkout main
git pull origin main


4) TAO BRANCH RIENG CHO TASK
----------------------------
Vi du lam login:
git checkout -b feature/login-quy

Hoac:
git checkout -b bugfix/fix-login-validation


5) CODE TRONG BRANCH RIENG
--------------------------
Sua file, them file, xoa file theo task.


6) KIEM TRA FILE DA DOI
-----------------------
git status
git diff


7) THEM FILE VAO STAGING
------------------------
Them tat ca:
git add .

Hoac them tung file:
git add src/.../LoginController.java
git add src/.../UserService.java


8) COMMIT
---------
git commit -m "add login API and validation"

Khuyen nghi:
- Moi commit nen nho, de review.
- Commit theo y nghia (1 commit = 1 nhom thay doi ro rang).


9) PUSH BRANCH LEN REMOTE
-------------------------
Lan dau:
git push -u origin feature/login-quy

Lan sau:
git push


10) TAO PULL REQUEST (PR)
-------------------------
Tren GitHub:
- Pull Requests -> New Pull Request
- base: main
- compare: feature/login-quy
- Title ro rang
- Description nen co:
  + Summary thay doi
  + Cach test
  + Anh/chung minh (neu co UI)
  + Link issue/task (neu co)

Vi du:
Title: Add login feature
Description:
- create login API
- validate input
- add login UI


11) REVIEW VA SUA THEO FEEDBACK
-------------------------------
Sau khi reviewer comment:
- Sua code tren cung branch feature
- Commit tiep
- Push lai

PR se tu dong cap nhat.


12) DONG BO BRANCH FEATURE VOI MAIN MOI NHAT (KHI PR CHUA MERGE)
-----------------------------------------------------------------
Khuyen nghi dung merge main vao branch feature:

git checkout main
git pull origin main
git checkout feature/login-quy
git merge main

Neu co conflict:
- Sua conflict trong file
- git add <file-da-sua>
- git commit
- git push

Ghi chu:
- Neu team dung rebase thi lam theo policy team.
- Neu chua thong nhat, uu tien merge de an toan.


13) MERGE PR
------------
Sau khi duoc approve va CI xanh:
- Merge Pull Request
- Confirm Merge

Khuyen nghi:
- Xoa branch tren GitHub sau khi merge (Delete branch).


14) CAP NHAT LAI MAY SAU KHI MERGE
----------------------------------
git checkout main
git pull origin main

Xoa branch local da merge:
git branch -d feature/login-quy

Neu branch chua merged ma van muon xoa:
git branch -D feature/login-quy


15) BAT DAU TASK MOI
--------------------
Lap lai quy trinh:
- update main
- tao branch moi
- code
- PR


BEST PRACTICES (NEN AP DUNG)
----------------------------
1. Luon pull main truoc khi tao branch.
2. Khong push truc tiep len main.
3. Moi PR nen nho, de review nhanh.
4. Commit thuong xuyen, message de hieu.
5. Khong de PR qua lon, qua lau moi merge.
6. Luon chay test/lint truoc khi push.
7. Neu co migration DB, ghi ro cach chay migration trong PR.
8. Neu sua config quan trong, note ro tac dong.


CHECKLIST TRUOC KHI TAO PR
--------------------------
- [ ] Da pull main moi nhat
- [ ] Da resolve conflict (neu co)
- [ ] Code build duoc
- [ ] Test/lint pass
- [ ] Khong commit file nhay cam (.env, key, token)
- [ ] PR co title + description ro rang


MOT SO LENH HAY DUNG NHANH
--------------------------
Xem lich su commit ngan:
git log --oneline --decorate --graph -20

Xem branch local:
git branch

Xem branch remote:
git branch -r

Xem file thay doi chua add:
git diff

Xem file da add:
git diff --staged

Huy thay doi chua add cua 1 file:
git restore <file>

Bo file khoi staging:
git restore --staged <file>


16) SETUP CI TU DONG (GITHUB ACTIONS)
-------------------------------------
Muc tieu:
- Moi PR vao main deu chay test/lint/build tu dong.
- Neu fail thi KHONG merge.

B1. Tao folder workflow:
- .github/workflows/

B2. Tao file:
- .github/workflows/ci.yml

Mau ci.yml don gian (Java Maven + Flutter):

name: CI
on:
  pull_request:
    branches: [ "main" ]
  push:
    branches: [ "main" ]

jobs:
  backend:
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: supermarket/supermarket
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: "21"
      - name: Build backend
        run: mvn -DskipTests compile

  frontend:
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: supermarket_Manager_System
    steps:
      - uses: actions/checkout@v4
      - uses: subosito/flutter-action@v2
        with:
          channel: stable
      - name: Install deps
        run: flutter pub get
      - name: Analyze
        run: flutter analyze
      - name: Test
        run: flutter test

Ghi chu:
- Sửa working-directory theo dung cau truc project cua repo.
- Neu co private package/secret thi them Secrets trong Settings > Secrets and variables > Actions.


17) BAT BRANCH PROTECTION CHO MAIN (BAT BUOC NEU MUON AN TOAN)
---------------------------------------------------------------
GitHub repo -> Settings -> Branches -> Branch protection rules -> Add rule:

Rule cho branch name pattern: main

Bat cac tuy chon:
- [x] Require a pull request before merging
- [x] Require approvals (vi du 1 hoac 2)
- [x] Dismiss stale approvals when new commits are pushed (khuyen nghi)
- [x] Require status checks to pass before merging
  -> Chon cac check tu workflow CI (backend, frontend)
- [x] Require branches to be up to date before merging
- [x] Restrict who can push to matching branches (khuyen nghi)
- [x] Do not allow bypassing the above settings (khuyen nghi)

Ket qua:
- Khong ai merge thang main khi CI do.
- Moi thay doi vao main bat buoc qua PR + review + CI xanh.


18) SETUP AUTO-MERGE PR VAO MAIN
--------------------------------
Auto-merge tren GitHub KHONG co nghia la merge vo dieu kien.
No chi merge khi DU DIEU KIEN:
- PR da duoc approve
- Tat ca required checks pass
- Branch da up-to-date voi main (neu bat rule)

Cach bat:
1. Repo -> Settings -> General
2. Bat:
   - [x] Allow auto-merge

Khi tao PR:
1. Bam "Enable auto-merge"
2. Chon kieu merge:
   - Squash and merge (khuyen nghi)
   - Hoac Rebase / Merge commit theo policy team

Tu do:
- Team khong can ngoi doi bam merge thu cong.
- Khi CI xong + du approval, GitHub se tu merge vao main.


19) TU DONG XOA BRANCH SAU KHI MERGE
------------------------------------
Repo -> Settings -> General
- [x] Automatically delete head branches

Loi ich:
- Repo sach se, giam branch rac.


20) QUY TRINH DE XUAT CHUAN CHO TEAM (CI + AUTO-MERGE)
------------------------------------------------------
1. Dev tao branch feature/*
2. Dev code + push
3. Tao PR vao main
4. CI tu chay (GitHub Actions)
5. Reviewer approve
6. Bat Enable auto-merge cho PR
7. GitHub tu merge khi tat ca dieu kien OK
8. Branch feature tu dong bi xoa (neu bat setting)


21) CHECKLIST KHI BAT DAU CI/AUTO-MERGE
---------------------------------------
- [ ] Co file .github/workflows/ci.yml
- [ ] CI chay xanh tren 1 PR thu nghiem
- [ ] Branch protection cho main da bat
- [ ] Required checks da chon dung ten job
- [ ] Repo da bat Allow auto-merge
- [ ] Team thong nhat kieu merge (Squash/Rebase/Merge commit)


22) LOI THUONG GAP
------------------
1. "Expected — Waiting for status to be reported"
   -> Ten job trong Branch protection khong khop voi ten job trong ci.yml

2. Auto-merge khong bat duoc
   -> Chua bat "Allow auto-merge" trong repo settings

3. PR khong merge du CI xanh
   -> Chua du so luong approvals, hoac branch chua up-to-date

4. CI fail do sai duong dan
   -> Kiem tra working-directory trong workflow
